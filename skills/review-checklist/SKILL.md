# Review Checklist

## Priority Order
1. Correctness bugs and regressions.
2. Security and data-loss risks.
3. Reliability and failure-mode gaps.
4. Missing or weak tests.
5. Maintainability concerns.

## Output Shape
- List findings first.
- Include severity and concrete file references.
- Keep summary short and secondary.
